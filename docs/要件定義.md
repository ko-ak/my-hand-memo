## 2. プロジェクト概要

### 2.1 背景

PCとAndroidの両方で手書きメモを取れるツールの開発。メモはGoogle Driveに保存し、オフライン時はローカルに一次保存して定期的に同期する。

### 2.2 目的

- マルチデバイス対応の手書きメモアプリの提供
- ペン・指・マウスの別操作認識による直感的な操作性の実現
- レイヤー機能や選択範囲機能による高度な編集機能の提供

## 3. 機能要件

### 3.1 基本機能

#### 3.1.1 手書き入力

| 機能 | 詳細 |
|------|------|
| ペン入力 | スタイラスペンでの描画を認識 |
| 指入力 | デフォルトはドラッグ（画面移動）。モード切替で描画可能 |
| マウス入力 | マウスでの描画を認識 |
| マルチタッチ | 2本指での拡大縮小・画面移動 |

#### 3.1.2 入力モード切替

| モード | 指の操作 | 2本指操作 |
|--------|----------|-----------|
| デフォルト | ドラッグ（画面移動） | 拡大縮小 |
| 描画モード | 描画 | 画面移動・拡大縮小 |

#### 3.1.3 テキスト入力

- キーボード操作によるテキスト入力が可能
- テキストとして保存可能

### 3.2 メモ管理機能

#### 3.2.1 メモ作成

| 機能 | 詳細 |
|------|------|
| タイトル自動設定 | デフォルトで「年月日_」を設定（例: 20260511_） |
| タイトル編集 | 画面上に表示し、タップで編集可能 |

#### 3.2.2 メモ保存

| 保存先 | タイミング | 詳細 |
|--------|-----------|------|
| ローカル | リアルタイム | 作成中のデータを一次保存 |
| Google Drive | 定期（5分おき等） | 更新があった場合に同期 |
| Google Drive | 手動 | 任意のタイミングで保存可能 |

#### 3.2.2.1 Google Drive連携の初期設定フロー

| ステップ | 操作 | 詳細 |
|---------|------|------|
| 1 | 初回起動時のプロンプト表示 | 「Google Driveに保存する」ボタンを表示 |
| 2 | Google認証画面への遷移 | ボタン押下でGoogle OAuth認証画面を開く |
| 3 | Googleアカウント選択 | ユーザーがGoogleアカウントを選択 |
| 4 | 権限付与 | Google Driveへのアクセス権限を付与（ファイルの読み書き権限） |
| 5 | アクセストークン取得 | アクセストークンとリフレッシュトークンを取得 |
| 6 | トークン保存 | トークンをローカルストレージに保存 |
| 7 | Google Driveフォルダ作成 | 「手書きメモ」フォルダをGoogle Driveに作成 |
| 8 | 設定完了 | 設定完了メッセージを表示し、通常のメモ作成画面へ遷移 |

**同期タイミング:**
- ローカル保存: リアルタイム（作成中のデータを一次保存）
- Google Drive保存: 定期（5分おき）に更新があった場合に同期
- 手動保存: 任意のタイミングで保存可能

**注意事項:**
- 初回設定は必須ではなく、ローカルのみでの使用も可能
- 設定は後からでも変更可能（設定画面からGoogle Drive連携の有効/無効切替）
- トークン有効期限切れの場合は、自動で再認証を促す
- オフライン時はローカル保存のみ行い、オンライン時に同期

#### 3.2.3 メモリスト

| 表示項目 | 詳細 |
|----------|------|
| タイトル | メモのタイトル |
| 作成日時 | メモの作成日時 |
| 更新日時 | メモの最終更新日時 |
| サムネイル | メモの内容をサムネイル表示 |
| 操作 | タップでメモを開く |

#### 3.2.3.1 メモリスト取得時の同期方法

**同期方法:**
- タイムスタンプ比較による増分同期を採用
- ローカルとGoogle Driveの更新日時を比較し、最新のメモを優先して表示
- Google Drive側のタイムスタンプを信頼し、ローカル側は補助的に使用

**増分同期の仕組み:**
- 前回同期時のタイムスタンプをローカルに保存
- Google Driveから「前回同期以降に更新されたメモ」のみをリクエスト
- 取得したメモとローカルのメモをタイムスタンプ比較
- 最新のメモを採用してリスト表示
- 最新の同期タイムスタンプを更新

**同期フロー:**
1. アプリ起動時にローカルのメモリストを即座に表示
2. Google Driveから前回同期以降のメモを増分取得
3. 取得したメモとローカルのメモをタイムスタンプ比較
4. 最新のメモでリストを更新
5. 最新の同期タイムスタンプを保存

**注意事項:**
- Google Driveのファイルコピー時は、新しいファイルIDが割り当てられるため、ローカルにはないファイルとして取得可能
- ファイルIDを基準に同期を行うことで、コピーされたファイルも正しく取得
- データ通信量を削減するため、増分同期を採用

## 4. 非機能要件

### 4.1 パフォーマンス

- 手書きの描画遅延を最小限に抑える（目標: 16ms以内）
- ローカル保存は即座に完了する

### 4.2 ユーザビリティ

- PCとAndroidで一貫した操作感を提供
- 直感的なUI/UXデザイン

### 4.3 セキュリティ

- Firebase Authenticationによる認証
- メールアドレスとパスワードでログイン
- アカウント登録はFirebase管理画面から行う（新規登録機能なし）

### 4.4 可用性

- オフライン時でもローカル保存で機能を提供
- Firebase同期失敗時はリトライ機能を提供

## 6. 質問事項

### 6.1 フレームワークの選定

PC/Android両対応とのことですが、どのようなアプローチを想定されていますか？

- **A案: React Native / Flutter（クロスプラットフォーム）**
  - メリット: 単一のコードベースで両対応、ネイティブに近いパフォーマンス
  - デメリット: 学習コスト、一部Web機能の制限

- **B案: PWA（Progressive Web App）**
  - メリット: Web技術一本化、開発効率が高い、PC/Android両対応
  - デメリット: ネイティブ機能へのアクセス制限、一部デバイスでの制限

- **C案: Web（PC）+ Androidネイティブ（Kotlin）**
  - メリット: 各プラットフォーム最適化、機能制限なし
  - デメリット: 開発コストが高い、コードの重複

**PM推奨**: PWAを推奨。理由として以下の通り。
- 開発効率が高く、早期MVP開発が可能
- 近年のPWAはネイティブに近い体験を提供
- 手書きメモアプリの要件はPWAで十分カバー可能
- 将来的にネイティブ化が必要になった場合でも移行が容易

→PWAでおねがいします。

### 6.2 手書きエンジンの選定

Canvas APIを使用する想定でよろしいでしょうか？それとも専用ライブラリ（Fabric.js, Konva.jsなど）の利用を検討しますか？

- **Canvas API**: 軽量、ブラウザ標準
- **Fabric.js / Konva.js**: 高度な機能（オブジェクト操作、レイヤー管理等）が充実

**PM推奨**: Konva.jsを推奨。理由として以下の通り。
- レイヤー機能、選択範囲機能など高度な編集機能が必要
- オブジェクトベースの操作が容易
- パフォーマンスが良好

→Konva.jsでおねがいします。

### 6.3 ローカル一次保存の方式

IndexedDB / localStorage / ファイルシステムのどれを想定されていますか？

- **IndexedDB**: 大容量対応、非同期処理、構造化データ
- **localStorage**: 小容量（約5MB）、同期処理、シンプル
- **ファイルシステム**: ファイルとして保存、ネイティブアプリに近い

**PM推奨**: IndexedDBを推奨。理由として以下の通り。
- 手書きデータは容量が大きくなる可能性があるため、大容量対応が必要
- 非同期処理により描画パフォーマンスに影響を与えない
- 構造化データとして管理しやすい

→IndexedDBでおねがいします。

### 6.4 MVP（実用最小限の製品）の優先順位

全機能を一度に実装するか、フェーズ分けして開発しますか？

- **フェーズ1**: 基本手書き + Firebase保存 + メモリスト
- **フェーズ2**: レイヤー機能
- **フェーズ3**: 選択範囲機能（投げ縄ツール等）

**PM推奨**: フェーズ分けを推奨。理由として以下の通り。
- 早期にユーザーフィードバックを得られる
- 開発リスクを分散できる
- 各フェーズで品質を確保しやすい

→フェーズ分けでおねがいします。
